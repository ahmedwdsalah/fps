### what is VBS and SBS?

- **VBS (Virtual Best Solver)** — oracle that always picks the true fastest algorithm per array. theoretical ceiling, impossible in practice
- **SBS (Single Best Solver)** — always pick one fixed algorithm for everything. naive baseline
- SBS on full dataset = heapsort (not timsort — even though timsort wins 85% of arrays, heapsort has lower total time because timsort's failures are catastrophic on random data)

### VBS-SBS gap — why this thesis exists

- VBS-SBS gap = **19.14%** — 19% room for improvement over "just always use heapsort"
- if gap was 0%, no point building a selector
- if gap was 50%+, trivial problem

### in the end:

- **93% gap closed** — strong result, near theoretical ceiling
- **76% accuracy sounds modest** but accuracy is misleading — the "errors" are mostly introsort↔heapsort confusion which costs <1% runtime
- the real metric is regret (1.62%) not accuracy (76%)
- remaining 7% gap likely requires hardware-level signals (cache behavior, branch prediction) that structural features can't capture
