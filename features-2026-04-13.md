### what are the 16 features?

- 16 structural properties extracted from each array before sorting
- all O(n)
- grouped into 5 categories: sortedness, spread, shape, uniqueness, local order

### feature list

| # | Feature | Range | What it measures |
|---|---------|-------|-----------------|
| 1 | `length_norm` | [0,1] | array size / largest array in dataset | ( it has its limition which is can give length_norm=1 for 1M array and 1K array), maybe imporve it and use log(n)
| 2 | `adjacent_sorted_ratio` | [0,1] | adjacent pairs already in order |
| 3 | `runs_ratio` | [0,1] | number of monotonic runs / n |
| 4 | `longest_run_ratio` | [0,1] | longest sorted run / n |
| 5 | `inversion_ratio` | [0,1] | out-of-order pairs (subsampled to 2000 for large arrays) |
| 6 | `duplicate_ratio` | [0,1] | fraction of non-unique values |
| 7 | `top1_freq_ratio` | [0,1] | most common value frequency |
| 8 | `top5_freq_ratio` | [0,1] | top 5 values cumulative frequency |
| 9 | `entropy_ratio` | [0,1] | Shannon entropy of 32-bin histogram, normalized |
| 10 | `dispersion_ratio` | [0,1] | std dev / range |
| 11 | `iqr_norm` | [0,1] | interquartile range / range |
| 12 | `mad_norm` | [0,1] | median absolute deviation / range |
| 13 | `skewness_t` | R | signed log-transformed skewness |
| 14 | `kurtosis_excess_t` | R | signed log-transformed excess kurtosis |
| 15 | `outlier_ratio` | [0,1] | fraction of values beyond 3 IQR |
| 16 | `mean_abs_diff_norm` | [0,1] | mean adjacent difference / range |

### why these groups?

- **sortedness** (adj_sorted, runs, longest_run, inversion) — directly predicts timsort advantage. high sortedness = timsort wins
- **uniqueness** (duplicate, top1_freq, top5_freq) — many duplicates changes algorithm behavior (e.g. QBIN50 transforms)
- **spread** (dispersion, iqr, mad) — how values distribute across the range
- **shape** (skewness, kurtosis, entropy) — distribution shape, randomness level
- **scale + local order** (length_norm, mean_abs_diff, outlier_ratio) — array size and step-by-step behavior

### which features actually matter? (v5 importance)

| Feature | Importance | Why |
|---------|-----------|-----|
| `length_norm` | 26.2% | array size is strongest signal — small arrays = noise, large arrays = clear winners |
| `top5_freq_ratio` | 18.4% | duplicate concentration changes sorting dynamics |
| `longest_run_ratio` | 8.0% | directly measures timsort's advantage |
| `duplicate_ratio` | 8.0% | uniqueness level |
| `entropy_ratio` | 7.5% | randomness — low entropy = structured = timsort |
| `runs_ratio` | 5.8% | number of sorted runs |
| rest | <5% each | diminishing returns |

### weak features

- `outlier_ratio` (~1%) — very low variance across datasets, almost always near 0
- `skewness_t` / `kurtosis_excess_t` (~3% each) — limited discriminative power between algorithms

### validation

- 214/214 tests passed
- zero NaN, zero Inf across all 1.18M arrays
- single source of truth: `scripts/feature_extraction.py` — imported by every training script
- one exception: `inversion_ratio` is O(n log n) not O(n), but subsampled to 2000 elements for arrays >10K so cost stays bounded
